1. Executive Summary
This Product Requirements Document defines the full scope, user experience, gamification design, and content architecture for SQUARES — a standalone, simulation-first, gamified web lesson for Grade 1 mathematics, specifically covering Lesson 6.2: Square from the Intellia SG curriculum.
The product mirrors the design ethos of the existing equal-tau.vercel.app lesson (Lesson 4.1: Equal Groups), applying the same React-based interactive pedagogy, audio narrative approach, and visual storytelling to introduce young learners to squares as a 2D geometric shape.
The lesson follows a strict Learn → Simulate → Practice → Celebrate pipeline. Every practice question is randomized per session, and every simulation interaction is visually rich and audio-guided. The product will be deployed directly on intelliasg.com/courses/grade-1-math/ under Lesson 6.2.
1.1 Mission Statement
To make every Grade 1 child in the world understand what a square is — not by memorization, but by seeing it, building it, touching it, and playing with it — through the most engaging 20-minute digital lesson they have ever experienced.
1.2 Key Differentiators
Simulation-first: children interact with shapes before answering questions
Audio-led narration matching the equal-tau.vercel.app voice style
100% randomized practice — no two sessions produce the same question order or values
Globally inclusive content referencing real-world square objects from 20+ countries
Matches Intellia SG visual brand: bold typography, coral/purple/blue palette, emoji-accented UI
Celebration/reward system with stars, confetti, and encouraging audio feedback
2. Product Overview
2.1 What is This Product?
SQUARES is a single-page, scroll-based, fully interactive web lesson built in React. It is one lesson within the Intellia SG Grade 1 Math course — specifically Lesson 6.2 of Section 6 (Geometry: Shapes). The lesson takes approximately 15–25 minutes to complete depending on pace.
The lesson consists of four major phases delivered in sequence:
LEARN — Story-driven introduction with animated simulation models
EXPLORE — Student-interactive simulation sandbox (drag, build, tap)
PRACTICE — Randomized, gamified question sets with audio feedback
CELEBRATE — Stars, rewards, and completion animation

2.3 Learning Objectives
By the end of this lesson, the student will be able to:
Name and recognize a square as a 2D shape
Identify that a square has exactly 4 sides, all of equal length
Identify that a square has exactly 4 corners (right angles)
Distinguish squares from other shapes (circles, triangles, rectangles)
Find squares in real-world objects (tiles, windows, book covers, flags)
Describe a square using simple language: 'A square has 4 equal sides'

5.6 Section 5 — Explore: Build a Square (Simulation)
PRIMARY SIMULATION MODULE — This is the core interactive experience.
Grid canvas appears: 6x6 dot grid
Student is given colored tiles (unit squares) in a drag-and-drop interface
Task: 'Drag the tiles to make a square! Make sure all sides are the same length.'
Snap-to-grid system: tiles snap cleanly to grid positions
Real-time side measurement labels appear as tiles are placed
If sides are unequal: gentle audio 'Hmm, check your sides!' + side highlights in red
When a valid square is formed: green glow + confetti burst + audio: 'You made a square!'
Challenge levels (unlock sequentially): 2x2 square → 3x3 square → 4x4 square
Visual ruler appears alongside showing equal length comparison
5.7 Section 6 — Explore: Spot the Square (Simulation)
Shape identification simulation — randomized each session.
Screen fills with 6–9 mixed 2D shapes: circles, triangles, rectangles, squares
All shapes are SVG-animated, colorful, bouncing gently
Task: 'Tap all the SQUARES!'
Each shape randomizes: size, color, rotation, position
Tap a square: ✅ green glow, check sound, score +1
Tap a non-square: ❌ gentle shake animation, 'That's not a square, keep looking!'
Session ends when all squares are found: fanfare + star reward
Minimum 2 squares always present; maximum 4; positions randomized
5.8 Section 7 — Practice Questions (Randomized)
5–7 questions per session, randomly selected from a question bank of 30+ questions across 6 types.
Question Type A: Multiple Choice — Identification
'Which of these is a square?' — 4 shape images shown; student taps the correct one. Shapes, colors, and sizes randomize.
Question Type B: Property Questions
'How many sides does a square have?' — Tap-to-select (number buttons 2, 3, 4, 5). Answer is always 4 but wrong options randomize.
Question Type C: Counting Corners
'Count the corners on this square' — Animated square shown; student taps each corner to count. Confetti when all 4 are tapped.
Question Type D: True or False
Statement shown: e.g. 'A square has 3 sides.' Student taps TRUE or FALSE. Statements randomize from a pool of 12.
Question Type E: Real-World Sorting
3–4 objects shown (mix of square and non-square items). Student drags each to correct bin: 'Square' or 'Not a Square'. Objects randomize from the 20+ item pool.
Question Type F: Equal Sides Comparison
Two shapes shown side by side. 'Which shape has all equal sides?' — reinforces core square property. Distractor shapes always include one with obviously unequal sides.
