1. System Architecture Overview
1.1 Architecture Pattern
The SQUARES lesson is a client-side Single Page Application (SPA) with zero backend requirements for the core lesson. All state is managed in-memory per session (React state). Optional persistence uses browser localStorage.
Architecture: Component-based React SPA → Vite build → Static CDN (Vercel)

1.2 Key Architectural Decisions
Decision
Rationale
No Backend
Lesson is fully static. No API calls during lesson. Randomization runs client-side with Math.random() + seeded shuffle (Fisher-Yates).
React + Vite
Matches equal-tau.vercel.app reference codebase (github.com/dsamyak/equal). Fast HMR, tree-shaking, modern ESM.
TypeScript
All components, state, and question data typed. Prevents runtime errors in production.
Framer Motion
Primary animation library. Replaces CSS-only animation for complex sequencing (stagger, drag, spring physics).
Tailwind CSS
Utility-first CSS matching reference repo structure. JIT mode for minimal bundle.
Web Audio API
Custom SFX generated programmatically (oscillator nodes) + preloaded MP3 narration. No audio library dependency.
SVG for Shapes
All geometric shapes (square, circle, triangle, rectangle) rendered as inline SVG for crisp scaling at any DPI.
localStorage
Optional session persistence. Lesson progress and star count saved per lesson ID.

5.2 Session Generation Algorithm
// src/engine/randomizer.ts
/**
 * Fisher-Yates shuffle — cryptographically weak but sufficient for educational randomization.
 * For true unpredictability per session, seed with Date.now() % 1000.
 */
function fisherYatesShuffle<T>(arr: T[]): T[] {
  const a = [...arr];
  for (let i = a.length - 1; i > 0; i--) {
    const j = Math.floor(Math.random() * (i + 1));
    [a[i], a[j]] = [a[j], a[i]];
  }
  return a;
}
/**
 * Generates a session question set:
 * - Session length: randomly 5, 6, or 7 questions
 * - Guaranteed 1x mcq-identification, 1x property-count, 1x corner-count
 * - Remaining slots: weighted random from remaining types
 * - Each question generated via its factory (values randomized within)
 */
export function generateSession(): GeneratedQuestion[] {
  const sessionLength = [5, 6, 7][Math.floor(Math.random() * 3)];
  
  // Core required questions (always included)
  const required: QuestionType[] = [
    'mcq-identification', 
    'property-count', 
    'corner-count'
  ];
  
  // Fill remaining slots from weighted pool
  const optional: QuestionType[] = [
    'true-false', 'true-false',          // weight 2
    'real-world-sort', 'real-world-sort', // weight 2
    'equal-sides-compare',                // weight 1
  ];
  
  const fillerCount = sessionLength - required.length;
  const shuffledOptional = fisherYatesShuffle(optional).slice(0, fillerCount);
  
  const allTypes = fisherYatesShuffle([...required, ...shuffledOptional]);
  
  return allTypes.map(type => {
    const question = QUESTION_BANK.find(q => q.type === type)!;
    return question.generate();  // Each generate() call produces fresh random values
  });
}

interface BuilderState {
  grid: GridCell[][];    // 6x6 grid
  placedTiles: PlacedTile[];

  isComplete: boolean;
  validation: ValidationResult;
}
interface ValidationResult {
  formed: boolean;
  sidesEqual: boolean;
  sideLength: number | null;
  incorrectSides: string[];  // IDs of sides that are wrong length
}
Grid validation algorithm:
On each tile placement, scan the grid for the largest contiguous rectangular region of occupied cells
Calculate bounding box of occupied region: width = columns span, height = rows span
Square condition: width === height AND all cells within bounding box are occupied
If width !== height: highlight the longer side in red + audio hint
Valid square: trigger completion animation + advance challenge level

// Dispatches to correct question component based on type
export function QuestionRenderer({ question, onAnswer }: Props) {
  const components: Record<QuestionType, React.FC<QuestionProps>> = {
    'mcq-identification': MCQQuestion,
    'property-count': PropertyQuestion,
    'corner-count': CornerCountQuestion,
    'true-false': TrueFalseQuestion,
    'real-world-sort': SortingQuestion,
    'equal-sides-compare': ComparisonQuestion,
  };
  
  const Component = components[question.type];
  
  return (
    <AnimatePresence mode="wait">
      <motion.div key={question.id} initial={{ opacity: 0, y: 30 }}
        animate={{ opacity: 1, y: 0 }} exit={{ opacity: 0, y: -30 }}>
        <Component question={question} onAnswer={onAnswer} />
      </motion.div>
    </AnimatePresence>
  );
}
